MovieGame

MovieGame is a web-based trivia game modeled after the parlor game Six Degrees of Kevin Bacon that tests the player's knowledge of movie casts. It was built with a Python backend using the Flask microframework, interfaces with TMDb's (The Movie Database) API to retrieve data, and stores user data in a Google Firestore database. The Bootstrap CSS framework was used on the frontend for styling and pure CSS for the scrolling banner on the landing page. A multiplayer feature and mobile app version is in production. Visit the MovieGame site and play yourself at six-degrees-of-hollywood.appspot.com

My involvement in this project was as tech lead of a four person team. I was the key decision maker and planner. I was deeply involved in all aspects of product design, implementation, and testing. I also wrote major portions of both the frontend and backend including the overall application layout, the scrolling banner in CSS, the API endpoints that interface with TMDb, and the portions of the program which interact with the Firestore database.

JukeBox

JukeBox is a web-based application which allows multiple users to collaborate on a shared playlist by adding songs to the playlist and upvoting or downvoting the songs on the playlist resulting in the most popular song playing next. The features of this app including user authentication and song search are accomplished through the Spotify API. It was built with a Python backend using the Flask microframework and SQLite database. AngularJS was used as the frontend framework.

I was a member of a five person team who completed this project in one iteration of the Software Development Life Cycle using the Agile Scrum methodology. I took part in the architectural design during the planning phase. I implemented the getNextSong functionality from the frontend to the backend and database and contributed to the playback functionality.

Lightweight Web Analysis Tool

This Lightweight Web Analysis Tool is built entirely in JavaScript and HTML. It analyzes data that is entered into a text box and returns useful information about that data.

I worked as a member of a three person team over six 2 week sprints using the Agile Scrum methodology. I performed the duties of Scrum Master for two of those sprints and worked closely with the product owner to develop this tool for the Pittsburgh-based company Viz. I contributed significant portions of the JavaScript and HTML code base for all functionalities of the tool.

Frogger AI

This Python program solves a Pygame implementation of the classic game of Frogger utilizing a Q-learning reinforcement learning algorithm. The trained frog here has learned through approximately 3 hours of training and 6000 deaths. The trained frog acheives 2 homes in a row regularly and 3 homes in a row on occassion. There are over 196K states enumerated to represent a frog-state. An untrained frog is noticably different after about 5-10 minutes of training in that it begins dodging cars regularly.
